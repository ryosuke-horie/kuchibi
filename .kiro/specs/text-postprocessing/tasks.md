# Implementation Plan

- [x] 1. (P) AppSettings にテキスト後処理設定プロパティを追加する
  - テキスト後処理のオン・オフ（デフォルト: 有効）のプロパティを追加する
  - UserDefaults に即座に永続化し、起動時に復元する
  - `resetToDefaults()` に新規プロパティを含める
  - 既存の AppSettingsTests を拡張し、新規プロパティのデフォルト値・永続化・復元・リセットを検証する
  - _Requirements: 3.1_

- [x] 2. (P) TextPostprocessor を新設する
- [x] 2.1 空白正規化機能を実装する
  - TextPostprocessing プロトコルを定義し、テキストを受け取って後処理済みテキストを返すインターフェースを作る
  - 先頭と末尾の空白を除去する
  - 日本語文字（ひらがな、カタカナ、漢字、全角記号）同士の間にある半角スペースを除去する
  - 英数字と日本語文字の間のスペースは保持する
  - 連続する半角スペースを1つに正規化する
  - 各変換ルールごとにユニットテストを追加する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 2.2 繰り返しテキスト除去機能を実装する
  - 3文字以上の連続する同一フレーズの繰り返しを検出し、1つに集約する
  - 2文字以下の繰り返しは除去しない
  - ユニットテストで繰り返し検出と保持の両方を検証する
  - _Requirements: 2.1, 2.2_

- [x] 3. SessionManager にテキスト後処理パイプラインを統合する
  - SessionManager の初期化時に TextPostprocessor を依存注入で受け取る
  - `lineCompleted` のハンドリングで、後処理が有効な場合のみテキスト変換を適用してから出力に渡す
  - 後処理が無効の場合は元のテキストをそのまま出力する
  - 既存の SessionManagerTests をプロトコル変更に合わせて更新する
  - _Requirements: 3.2, 3.3_

- [x] 4. SettingsView にテキスト後処理設定 UI を追加する
  - 「音声認識」タブの前処理セクションの後にテキスト後処理の Toggle を配置する
  - _Requirements: 3.1_
