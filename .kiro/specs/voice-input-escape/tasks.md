# Implementation Plan

- [x] 1. ESCキーグローバル監視コンポーネントの実装
- [x] 1.1 EscapeKeyMonitoring プロトコルとテスト用 Mock を定義する
  - ESC キー監視の開始・停止を抽象化するプロトコルを定義する
  - 監視開始時にコールバックを受け取り、ESC 検出時にそのコールバックを呼び出す設計とする
  - ユニットテストで利用できる MockEscapeKeyMonitor を合わせて作成する
  - _Requirements: 1.4_

- [x] 1.2 EscapeKeyMonitorImpl を実装する（NSEvent グローバルモニター）
  - NSEvent のグローバルイベントモニターを用いて ESC キー（keyCode 53）を検出する
  - イベントを横取りせず監視のみ行う実装とする（他アプリの ESC 操作に干渉しない）
  - アプリ起動から終了まで常時モニターを維持し、二重登録を防止する
  - stopMonitoring 呼び出し後は ESC を押してもコールバックが呼び出されないことを保証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 2. (P) セッションキャンセル機能の実装（タスク1と並行実行可能）
- [x] 2.1 (P) キャンセルサウンドIDとキャンセル処理のロジックを SessionManagerImpl に追加する
  - 通常終了音と区別できるキャンセル専用のシステムサウンドIDを既存サウンド定義に追加する（Basso 相当）
  - recording または processing 状態のときのみキャンセルを受け付け、idle 状態では何も行わない
  - 進行中の録音タスクを協調的にキャンセルし、音声キャプチャを即時停止する
  - 蓄積済みテキスト行と部分テキストをすべて破棄し、テキスト出力メソッドを呼び出さない
  - 音量レベルをゼロにリセットし、状態をアイドルに戻す
  - セッションサウンド設定が有効な場合のみキャンセルサウンドを再生する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4, 3.5, 4.1, 4.2_

- [x] 3. アプリへの統合（タスク1・2の完了後に実施）
  - KuchibiApp の初期化処理で EscapeKeyMonitorImpl を生成する
  - ESC 検出コールバック内でメインアクターへの切り替えを行い cancelSession を呼び出す
  - アプリ起動時に監視を開始し、既存のホットキー登録と同じライフサイクルで管理する
  - FeedbackBar はアイドル状態への遷移で自動非表示となるため追加実装は不要
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 4.3_

- [x] 4. テストの実装
- [x] 4.1 cancelSession() のユニットテストを実装する
  - recording 状態での cancelSession() 呼び出し後に状態がアイドルになることを検証する
  - processing 状態での cancelSession() 呼び出し後に状態がアイドルになることを検証する
  - idle 状態での cancelSession() 呼び出しがテキスト出力を行わないことを検証する
  - キャンセル後に蓄積テキストと部分テキストが空になることを検証する
  - キャンセル後に音声キャプチャ停止が呼ばれていることを MockAudioCaptureService で検証する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 4.2 (P) ESCキーキャンセルの統合テストを実装する
  - MockEscapeKeyMonitor を用いて ESC 検出から cancelSession 呼び出しまでの一連のフローを検証する
  - キャンセル後に startSession() が正常に動作することを確認する（状態復帰の検証）
  - MockOutputManager を用いてキャンセル時にテキスト出力が一切行われないことを確認する
  - _Requirements: 1.4, 3.5, 2.4_
