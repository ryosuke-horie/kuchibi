# Implementation Plan

- [ ] 1. プロジェクト基盤のセットアップ
- [ ] 1.1 Xcodeプロジェクトとパッケージ構成の初期化
  - Swift Package ManagerベースのmacOSアプリケーションプロジェクトを作成する
  - SPM依存としてMoonshine v2パッケージとHotKeyパッケージを追加する
  - macOS 14以上をデプロイメントターゲットに設定する
  - Info.plistで`LSUIElement`を`YES`に設定し、Dockアイコンを非表示にする
  - `NSMicrophoneUsageDescription`をInfo.plistに追加してマイク権限要求の説明文を設定する
  - エラー型`KuchibiError`の列挙型と、`SessionState`・`OutputMode`の共通型定義を作成する
  - _Requirements: 1.2_

- [ ] 1.2 メニューバーアプリのシェル構築
  - SwiftUIの`MenuBarExtra`シーンを使用してメニューバー常駐アイコンを表示するアプリエントリポイントを作成する
  - メニューバーアイコンクリック時に表示される状態メニュー（音声認識の状態表示、出力モード切替、終了ボタン）を実装する
  - メインウィンドウ（WindowGroup）を含めず、メニューバーのみで動作する構成にする
  - `SMAppService`を使用してログイン項目への自動起動登録機能を状態メニューに追加する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [ ] 2. 音声キャプチャ機能の実装
- [ ] 2.1 (P) マイク音声のリアルタイムキャプチャ
  - `AVAudioEngine`を使用して、マイク入力からPCMバッファをリアルタイムで取得する音声キャプチャサービスを実装する
  - 入力ノードにタップを設置し、音声データを`AsyncStream<AVAudioPCMBuffer>`として提供する
  - キャプチャの開始・停止を制御するインターフェースを実装する
  - マイク権限の確認・要求処理を実装し、権限が未付与の場合はユーザーに許可を促す
  - キャプチャ中は単一のオーディオタップのみが有効であることを保証する
  - _Requirements: 3.4, 3.5_
  - _Contracts: AudioCaptureService_

- [ ] 3. 音声認識エンジンの統合
- [ ] 3.1 (P) Moonshineモデルアダプターの実装
  - Moonshine SPMパッケージを使用して、日本語モデル（`moonshine-tiny-ja`）の読み込み・初期化を行うアダプターを実装する
  - 日本語モデル用に`max_tokens_per_second`を13.0に設定する
  - 音声バッファを逐次投入する`addAudio`、途中結果を取得する`getPartialText`、最終結果を確定する`finalize`のインターフェースを実装する
  - モデル読み込み失敗時に適切なエラーをスローする
  - _Requirements: 3.1, 3.2, 3.3_
  - _Contracts: MoonshineAdapter_

- [ ] 3.2 音声認識サービスの構築
  - MoonshineAdapterを使用して、音声ストリームを受け取りテキスト認識イベント（開始・途中更新・完了）を発行するサービスを実装する
  - `AsyncStream<AVAudioPCMBuffer>`を入力として受け取り、`AsyncStream<RecognitionEvent>`を出力する変換パイプラインを構築する
  - すべての認識処理がローカルで完結し、外部通信を行わないことを保証する
  - モデル読み込み済みかどうかの状態管理を実装する
  - 3.1のMoonshineAdapterに依存
  - _Requirements: 3.1, 3.2, 3.3, 5.1_
  - _Contracts: SpeechRecognitionService_

- [ ] 4. テキスト出力機能の実装
- [ ] 4.1 (P) クリップボード操作と直接入力サービス
  - `NSPasteboard`を使用してテキストをシステムクリップボードにコピーする機能を実装する
  - `CGEvent`を使用してCmd+Vキーストロークをシミュレートし、アクティブアプリにテキストを直接入力する機能を実装する
  - 直接入力時にユーザーの元のクリップボード内容を退避し、ペースト後に復元する処理を実装する
  - Accessibility権限が未付与の場合の検出と通知処理を実装する
  - _Requirements: 4.1, 4.2, 4.4_
  - _Contracts: ClipboardService_

- [ ] 4.2 出力モード管理
  - クリップボードコピーモードと直接入力モードを切り替えて認識結果を出力するマネージャーを実装する
  - 選択された出力モードをUserDefaultsに永続化し、アプリ再起動後も設定を保持する
  - ClipboardServiceを使用して、モードに応じた出力処理を実行する
  - 4.1のClipboardServiceに依存
  - _Requirements: 4.1, 4.2, 4.4_
  - _Contracts: OutputManager_

- [ ] 5. グローバルホットキー制御の実装
- [ ] 5.1 (P) ホットキー登録とトグル制御
  - HotKeyライブラリを使用して、グローバルホットキー（デフォルト: Cmd+Shift+Space）を登録する
  - ホットキー押下時にトグル方式で音声入力セッションの開始・停止を切り替えるコントローラーを実装する
  - 他のアプリケーションがフォーカスされている状態でもホットキーを受け付けることを確認する
  - アプリ起動時にホットキーを登録し、終了時に解除する
  - _Requirements: 2.1, 2.2, 2.3_
  - _Contracts: HotKeyController_

- [ ] 6. セッション管理とパイプライン統合
- [ ] 6.1 セッションマネージャーの実装
  - 音声入力セッションのライフサイクル（Idle → Recording → Processing → Idle）を管理するセッションマネージャーを実装する
  - AudioCaptureService → SpeechRecognitionService → OutputManager のパイプラインをオーケストレーションする
  - セッション開始時に音声キャプチャを開始し、認識イベントのストリーミング処理を開始する
  - セッション停止時に音声キャプチャを停止し、最終認識結果を出力する
  - 認識途中テキスト（partialText）をObservableObjectとしてUI層に公開する
  - MainActor上で状態更新を行い、音声処理は非同期タスクで実行する
  - タスク2, 3, 4, 5の各サービスに依存
  - _Requirements: 2.1, 2.2, 4.3, 5.2, 5.3, 5.4_
  - _Contracts: SessionManager_

- [ ] 6.2 ホットキーとセッションの接続
  - HotKeyControllerのイベントをSessionManagerに接続し、ホットキー押下でセッション開始・停止が実行されるようにする
  - SessionManagerの状態に基づいてトグル動作（idle時→開始、recording時→停止）を制御する
  - 6.1のSessionManagerに依存
  - _Requirements: 2.1, 2.2_

- [ ] 7. 録音オーバーレイUIの実装
- [ ] 7.1 録音インジケーターとストリーミングテキスト表示
  - 音声入力セッション中に表示されるフローティングオーバーレイウィンドウをSwiftUIで実装する
  - 録音中であることを視覚的に示すインジケーター（アニメーション付きアイコン等）を表示する
  - SessionManagerから受け取った認識途中テキスト（partialText）をリアルタイムで表示する
  - セッション終了時にオーバーレイを自動的に非表示にする
  - 他のウィンドウの上に常に表示され、フォーカスを奪わないウィンドウレベルの設定を行う
  - 6.1のSessionManagerに依存
  - _Requirements: 2.4, 4.3_
  - _Contracts: RecordingOverlay_

- [ ] 8. エラーハンドリングと通知フィードバック
- [ ] 8.1 通知サービスとエラーフィードバック
  - UserNotificationsフレームワークを使用してmacOS通知センターにエラー通知を送信するサービスを実装する
  - モデル読み込みエラー、マイク権限エラー、認識エラーの各ケースに対応した通知内容を定義する
  - マイク権限・Accessibility権限が未付与の場合にシステム環境設定を開く誘導通知を実装する
  - _Requirements: 5.1, 5.4_
  - _Contracts: NotificationService_

- [ ] 8.2 タイムアウトと完了フィードバック
  - SessionManagerに無音タイムアウト監視を追加し、一定時間音声が検出されない場合にセッションを自動終了する
  - 音声認識が正常に完了した場合のサウンドまたは視覚的フィードバックを実装する
  - 認識処理中にエラーが発生した場合のセッション安全終了処理を実装する
  - 8.1の通知サービスに依存
  - _Requirements: 5.2, 5.3, 5.4_

- [ ] 9. エンドツーエンド統合と動作検証
- [ ] 9.1 全コンポーネントの統合配線
  - アプリエントリポイント（KuchibiApp）から全サービスのインスタンス生成と依存注入を行う
  - メニューバーUIにSessionManagerの状態をバインドし、出力モード切替を接続する
  - HotKeyController → SessionManager → 各サービスの完全なイベントフローを接続する
  - すべてのタスクに依存
  - _Requirements: 1.1, 1.3, 2.1, 2.2_

- [ ] 9.2 統合動作テスト
  - ホットキー押下 → 音声録音 → テキスト認識 → クリップボードコピーの一連のフローを検証する
  - ホットキー押下 → 音声録音 → テキスト認識 → アクティブアプリへの直接入力の一連のフローを検証する
  - 無音タイムアウトによるセッション自動終了とエラー通知の動作を検証する
  - メニューバーの状態表示が各セッション状態と連動していることを確認する
  - _Requirements: 1.1, 2.1, 2.2, 3.1, 3.2, 4.1, 4.2, 5.2, 5.3_
