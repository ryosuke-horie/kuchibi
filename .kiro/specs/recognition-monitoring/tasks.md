# Implementation Plan

- [x] 1. (P) AppSettings にモニタリング設定プロパティを追加する
  - モニタリング有効フラグ（デフォルト: 有効）のプロパティを追加する
  - UserDefaults に即座に永続化し、起動時に復元する
  - `resetToDefaults()` に新規プロパティを含める
  - 既存の AppSettingsTests を拡張し、デフォルト値・永続化・復元・リセットを検証する
  - _Requirements: 3.1_

- [x] 2. (P) SessionMonitoringService を新設する
  - セッションモニタリングのプロトコルを定義し、セッション開始・テキスト確定・セッション完了・エラー終了の各メソッドを持つインターフェースを作る
  - セッション開始時に開始時刻を記録し、内部の行数・文字数カウンタをリセットする
  - テキスト確定ごとに行数を加算し、文字数を累計する
  - セッション完了時に継続時間を算出し、継続時間・行数・文字数のサマリーログを info レベルで出力する
  - エラー終了時にエラー種別と継続時間を error レベルでログ出力する
  - 各メソッドの動作をユニットテストで検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2_

- [x] 3. SessionManager にモニタリングサービスを統合する
  - SessionManager の初期化時に SessionMonitoring を依存注入で受け取る
  - セッション開始時にモニタリングが有効であればセッション開始を通知する
  - テキスト確定時にモニタリングが有効であれば確定テキストを通知する
  - セッション完了時にモニタリングが有効であればセッション完了を通知する
  - エラー終了時にモニタリングが有効であればエラーを通知する
  - モニタリング無効時にはモニタリングメソッドを一切呼び出さない
  - 既存の SessionManagerTests をプロトコル変更に合わせて更新する
  - _Requirements: 3.2_

- [x] 4. SettingsView にモニタリング設定 UI を追加する
  - 「後処理」セクションの後にモニタリングの Toggle を配置する
  - _Requirements: 3.3_
