# Implementation Plan

- [x] 1. フィラー除去のテストを追加する
  - 各フィラーパターン（長音系・複合系・短縮系・副詞的）が正しく除去されることを検証するテストを追加する
  - 文頭フィラーが除去され残りテキストが出力されることをテストする
  - 文中フィラーが除去され前後テキストが自然に接続されることをテストする
  - フィラーのみのテキストが空文字列を返すことをテストする
  - 意味のある語の一部（「あのね」「まあまあ」等）が誤除去されないことをテストする
  - フィラーと既存ルール（スペース正規化・繰り返し除去）の複合処理をテストする
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 2. フィラー除去ステップを後処理パイプラインに実装する
  - 日本語フィラーパターンを正規表現で定義する
  - 連続スペース正規化と日本語文字間スペース除去の間にフィラー除去ステップを挿入する
  - 単独出現のフィラーのみを除去し、他の語の一部として出現する場合は保持する
  - 全テスト（新規・既存）がパスすることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1_

- [x] 3. ビルドと全テスト実行による統合検証
  - プロジェクト全体をビルドしてコンパイルエラーがないことを確認する
  - 全テストを実行して既存機能にリグレッションがないことを確認する
  - 既存の後処理有効/無効テストでフィラー除去も制御されることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2_
