# Implementation Plan

- [x] 1. テキスト蓄積バッファの導入と lineCompleted ハンドラの変更
- [x] 1.1 蓄積バッファの追加とセッション開始時の初期化
  - SessionManager にテキスト蓄積用のプライベート配列プロパティを追加する
  - セッション開始時にバッファを空に初期化する
  - テスト: セッション開始後にバッファが空であることを、セッション終了時の出力が空であることで間接的に検証する
  - _Requirements: 1.3_

- [x] 1.2 lineCompleted ハンドラをバッファ蓄積方式に変更
  - lineCompleted イベント発生時に OutputManager への即時出力を除去する
  - 後処理が有効な場合は処理済みテキストをバッファに追加する
  - 後処理が無効な場合は元テキストをそのままバッファに追加する
  - モニタリング通知（`textCompleted`）は行単位で維持する
  - テスト: lineCompleted 発生時点で OutputManager が呼ばれていないことを検証する
  - テスト: 後処理有効時に処理済みテキストがバッファに蓄積されることを、セッション終了後の出力内容で検証する
  - テスト: 後処理無効時に元テキストがバッファに蓄積されることを、セッション終了後の出力内容で検証する
  - _Requirements: 1.1, 1.2, 3.1, 3.2_

- [x] 2. セッション終了時の一括出力
- [x] 2.1 finishSession でバッファをフラッシュ出力する
  - `finishSession()` を async に変更し、蓄積テキストを改行で結合して OutputManager に出力する
  - バッファが空の場合は OutputManager を呼び出さない
  - 出力後にバッファをクリアする
  - `finishSession()` の既存の呼び出し箇所すべてに `await` を追加する
  - テスト: 単一の lineCompleted 後にセッション終了で結合テキストが正しく出力されることを検証する
  - テスト: 複数の lineCompleted 後にセッション終了で改行結合されたテキストが出力されることを検証する
  - テスト: lineCompleted が発生せずセッション終了した場合に OutputManager が呼ばれないことを検証する
  - _Requirements: 2.1, 2.2_

- [x] 2.2 無音タイムアウト時の蓄積テキスト出力を検証する
  - 無音タイムアウトで `finishSession(error:)` が呼ばれた際にも蓄積テキストがフラッシュされることを確認する
  - テスト: 無音タイムアウト発生時に、それまでに蓄積されたテキストが OutputManager に出力されることを検証する
  - _Requirements: 2.3_

- [x] 3. 既存テストの更新と回帰確認
  - 既存の SessionManagerTests で蓄積方式に合わなくなったテストを修正する
  - `outputCalledAfterStop` テストは蓄積後の一括出力を検証するように調整する
  - 全テストスイートが通過することを確認する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 3.1, 3.2_
